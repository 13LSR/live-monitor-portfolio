# 🚀 项目需求文档 (PRD)：极客风实时监控活体作品集 (Live-Monitor Portfolio)

## 1. 项目概述 (Project Overview)

本项目旨在开发一个高度现代化、极具视觉冲击力的个人作品集网站。打破传统作品集仅展示静态截图的局限，本项目将采用 **“Bento Grid（便当盒）网格布局”** 结合 **“Iframe 实时微缩渲染”** 技术，让所有展示的 Web 应用程序以“活体”形式在页面中实时运行。同时，深度集成 **UptimeFlare** 监控服务，将作品集打造成一个兼具展示与运维属性的“极客监控大屏”。

## 2. 技术栈选型 (Tech Stack)

* **前端框架：** Next.js (React) + App Router
* **样式方案：** Tailwind CSS
* **UI/动效组件库：** Aceternity UI (负责发光边框、极客风动效、Bento 网格底层)
* **后端/状态监控：** UptimeFlare (基于 GitHub Actions & Cloudflare Workers)
* **部署环境：**
* 主站承载：Cloudflare Pages (全静态导出或 Edge 渲染)
* 监控 API：Cloudflare Workers
* 子项目：Vercel / Cloudflare Pages / Render (需配合配置跨域/嵌套响应头)



## 3. 核心功能需求 (Core Features)

### 3.1 活体网页渲染 (Live Iframe Rendering)

* **机制：** 在 Bento 卡片中嵌套 `<iframe>`，强制以 PC 端分辨率（如 1920x1080）加载目标网站。
* **CSS 魔法：** 使用 `transform: scale()` 将 iframe 等比缩小至卡片尺寸，原点设为 `transform-origin: top left`。
* **性能与交互优化：** * 所有 iframe 必须配置 `loading="lazy"` 以实现懒加载，防止首屏内存溢出。
* 默认状态下设置 `pointer-events: none` 防止滚动穿透，仅在特定交互（如点击“进入交互模式”按钮）后开启。



### 3.2 实时状态监控 (Real-time Status Sync)

* **数据源：** 异步抓取 UptimeFlare 暴露的 `/api/data` 接口。
* **UI 联动：** * 提取每个子站点的 `up` (在线状态) 和 `latency` (延迟/Ping值)。
* 在线时：卡片状态栏显示绿色呼吸灯及真实延迟数据。
* 离线时：卡片状态栏显示红色警示灯，iframe 区域叠加一层带有高斯模糊和暗色遮罩的错误状态 UI。



### 3.3 跨域与嵌套安全策略 (CORS & CSP Bypass)

* **前端跨域获取 API：** 开发环境使用代理方案，生产环境在 UptimeFlare 仓库中配置 `_headers` 开放 `Access-Control-Allow-Origin: *`。
* **子站嵌套授权：** 所有受监控的云端子站（如 `wyy.331106.xyz` 等），必须在其部署平台的配置（如 Vercel 的 `vercel.json` 或 CF 的 `_headers`）中注入 `Content-Security-Policy: frame-ancestors 'self' [主站域名]`。

## 4. 页面模块规划 (Bento Grid Layout Planning)

整个页面由不规则的 Bento 卡片构成，根据内容属性进行差异化展示：

### 模块 A：主核心在线服务大卡片 (Web Services)

* 占据视觉中心的最大尺寸卡片。
* **内容：** 你的核心部署项目（例如已跑通测试的 `wyy.331106.xyz` 等独立域名站点）。
* **展示形态：** 完整的终端风格顶部栏 + 绿色 Ping 值指示灯 + 实时渲染的微缩网页。

## 5. 部署与实施路径 (DevOps Pipeline)

1. **后端基建验收：** 确保 `am-uptime-flare` 在 Cloudflare Workers 稳定运行，且已正确添加所需监控的所有独立域名。
2. **前端本地开发：** 基于 Next.js 初始化，引入 Tailwind 和 Aceternity UI，编写通用 `LiveProjectCard` 组件。
3. **内容填充与网格排版：** 使用 CSS Grid 或 Tailwind 的 `grid-cols` 构建 Bento 布局，将 模块 A 拼装入格。
4. **安全协议下发：** 前往 Vercel/Render 等平台，为所有目标子站配置允许嵌套的响应头。
5. **一键发布：** 将 Next.js 代码推送到 GitHub，在 Cloudflare Pages 面板中关联该仓库，执行构建命令 `npm run build`，完成全云端 Serverless 部署。
