3. FRONTEND PAGE SPLIT

Public page (/):
- Fetch site list from Worker GET /sites
- Fetch monitor data directly from UptimeFlare /api/data
- Render Bento layout
- Show live iframe cards for deployed sites
- Show only website portfolio cards and related empty-state messaging

Admin page (/admin):
- Unauthenticated state: password form only
- Authenticated state: site table + add/edit form + delete action
- Session check on load via Worker GET /auth/me

Important frontend rules:
- Keep app/page.tsx small; compose from _components
- Use "use client" only where iframe interaction, forms, state, or effects are needed
- Do not place secrets in Pages env vars
- Default public cards to pointer-events: none
- Allow opening target site in a new tab


4. DATA MODEL (D1)

Table: sites

Columns:
- id INTEGER PRIMARY KEY AUTOINCREMENT
- name TEXT NOT NULL
- slug TEXT NOT NULL UNIQUE
- url TEXT NOT NULL
- monitor_key TEXT
- category TEXT NOT NULL DEFAULT 'web'
- description TEXT DEFAULT ''
- cover_mode TEXT NOT NULL DEFAULT 'iframe'
- sort_order INTEGER NOT NULL DEFAULT 0
- is_visible INTEGER NOT NULL DEFAULT 1
- allow_interaction INTEGER NOT NULL DEFAULT 0
- created_at TEXT NOT NULL
- updated_at TEXT NOT NULL

Recommended meanings:
- slug: stable identifier for routing or UI keys
- monitor_key: UptimeFlare monitors object key, for example wyy.331106.xyz
- category: web, landing, tool, other
- cover_mode: iframe, text, gallery
- allow_interaction: whether UI can enable pointer interaction mode

6. WORKER API CONTRACT

Authentication:

POST /auth/login
Request:
{
  "password": "string"
}
Response 200:
{
  "ok": true
}
Behavior:
- Validate password against Worker secret ADMIN_PASSWORD
- Set signed HttpOnly session cookie

POST /auth/logout
Response 200:
{
  "ok": true
}
Behavior:
- Clear session cookie

GET /auth/me
Response 200:
{
  "authenticated": true
}
or
{
  "authenticated": false
}

Sites:

GET /sites
Response 200:
{
  "sites": [
    {
      "id": 1,
      "name": "Example Site",
      "slug": "example-site",
      "url": "https://example.com",
      "monitorKey": "example.com",
      "category": "web",
      "description": "Short text",
      "coverMode": "iframe",
      "sortOrder": 10,
      "isVisible": true,
      "allowInteraction": false,
      "createdAt": "2026-06-20T10:00:00.000Z",
      "updatedAt": "2026-06-20T10:00:00.000Z"
    }
  ]
}
Behavior:
- Public read is acceptable for this project
- Return only visible sites unless admin mode is needed

POST /sites
Auth required
Request:
{
  "name": "string",
  "slug": "string",
  "url": "https://...",
  "monitorKey": "string",
  "category": "web",
  "description": "string",
  "coverMode": "iframe",
  "sortOrder": 0,
  "isVisible": true,
  "allowInteraction": false
}

PUT /sites/:id
Auth required
Request body same shape as POST /sites

DELETE /sites/:id
Auth required
Response:
{
  "ok": true
}

Phase 2 optional endpoint:

GET /status
Response:
{
  "monitors": {
    "example.com": {
      "up": true,
      "latency": 122
    }
  }
}
Behavior:
- Worker fetches UptimeFlare and returns normalized data
- Add short cache if request volume becomes a problem


7. AUTH DESIGN

Chosen approach:
- Single admin password only
- Password stored in Worker secret: ADMIN_PASSWORD
- Cookie signing secret stored in Worker secret: SESSION_SECRET

Session flow:
1. Admin submits password from /admin
2. Worker validates password
3. Worker returns Set-Cookie with HttpOnly, Secure, SameSite=Lax
4. Frontend calls GET /auth/me to confirm logged-in state
5. Admin CRUD requests include cookie automatically

Do not do:
- Do not validate password in frontend code
- Do not store raw password in localStorage
- Do not expose ADMIN_PASSWORD as NEXT_PUBLIC_ env var

Pages deployment:
- Build command: npm run build
- Output directory: out
- next.config.ts must use output: 'export'
- Deploy via GitHub Actions to Cloudflare Pages

Worker deployment:
- Deploy from /worker using Wrangler
- Bind D1 database in wrangler.toml
- Store secrets in Cloudflare Worker settings or CI secret setup

GitHub repository secrets:
- CLOUDFLARE_API_TOKEN
- CLOUDFLARE_ACCOUNT_ID
