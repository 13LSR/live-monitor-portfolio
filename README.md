# Live Monitor Portfolio

A static-first Next.js frontend plus a Cloudflare Worker backend for managing and showcasing real deployed websites as live iframe portfolio cards.

## Local Development

Frontend only:

```bash
npm ci
npm run dev
```

Full local flow:

```bash
copy .env.example .env.local
npm run dev
cd worker
npm install
copy .dev.vars.example .dev.vars
npx wrangler dev
```

Expected local addresses:

- Frontend: `http://localhost:3000`
- Worker: `http://localhost:8787`

## Environment Variables

Frontend `.env.local`:

- `NEXT_PUBLIC_API_BASE_URL`
- `NEXT_PUBLIC_UPTIMEFLARE_API_URL`

Worker secrets:

- `ADMIN_PASSWORD`
- `SESSION_SECRET`

For local `wrangler dev`, put these in `worker/.dev.vars` or `worker/.env`.
Remote `wrangler secret put ...` values are not enough for local development.

## D1 Setup

Run these yourself when you're ready to bind real local and remote data:

```bash
cd worker
npx wrangler d1 create live-monitor-portfolio
```

Copy the returned `database_id` into `worker/wrangler.toml`, then run:

```bash
cd worker
npx wrangler secret put ADMIN_PASSWORD
npx wrangler secret put SESSION_SECRET
npx wrangler d1 migrations apply live-monitor-portfolio --local
```

For local Worker auth to work, also create:

```bash
cd worker
copy .dev.vars.example .dev.vars
```

Then edit `worker/.dev.vars` and set the same local password you want to use in `/admin`.

If you want to seed the default demo site after logging into `/admin`, you can call:

```bash
curl -X POST http://127.0.0.1:8787/sites/seed
```

The seed endpoint requires a valid admin session cookie, so in practice you should first log in from the browser-based `/admin` page, or seed manually by inserting your own sites through the admin form.

## Build Checks

Run these before any deployment:

```bash
npm run lint
npm run build
```

## Deployment

- `Cloudflare Pages` serves the static frontend generated by `next build`
- `Cloudflare Worker` handles auth and site CRUD
- GitHub Actions workflows live in `.github/workflows/`

Cloudflare-specific resources like D1 creation, secret injection, and dashboard credentials should be configured outside the repo.
